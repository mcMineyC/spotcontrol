## Quick Configuration (`QuickConfig`)

The `quick.Connect()` function accepts a `QuickConfig` struct that provides sensible defaults for most fields. This is the simplest way to configure spotcontrol.

```go
result, err := quick.Connect(ctx, quick.QuickConfig{
    StatePath:    "spotcontrol_state.json",
    DeviceName:   "MyApp",
    DeviceType:   spotcontrol.DeviceTypeComputer,
    DeviceId:     "",       // auto-generated or loaded from state
    Interactive:  true,     // enable OAuth2 PKCE login
    CallbackPort: 0,        // random available port
    Log:          nil,      // default: NewSimpleLogger(nil)
})
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `StatePath` | `string` | `""` | Path to the JSON state file for persistence. If empty, no state is saved or loaded. |
| `DeviceName` | `string` | `"SpotControl"` | Human-readable name shown in Spotify Connect device lists on other clients. |
| `DeviceType` | `spotcontrol.DeviceType` | `DeviceTypeComputer` | The device type, which affects how this controller appears in Spotify Connect device lists. |
| `DeviceId` | `string` | `""` (auto) | 40-hex-char device identifier. If empty and `StatePath` is set, loaded from state. If still empty, auto-generated by the session layer. |
| `Interactive` | `bool` | `false` | Enable OAuth2 PKCE browser-based login when no stored credentials are available. If `false` and no credentials exist, `Connect` returns an error. |
| `CallbackPort` | `int` | `0` | Port for the local OAuth2 callback HTTP server. `0` means a random available port is chosen. |
| `Log` | `spotcontrol.Logger` | `NewSimpleLogger(nil)` | Logger instance. If `nil`, a `SimpleLogger` writing to `os.Stderr` is created. |

### Defaulting Behavior

The `ApplyDefaults` function fills in zero-valued fields:

```go
func ApplyDefaults(cfg QuickConfig) QuickConfig {
    if cfg.Log == nil {
        cfg.Log = spotcontrol.NewSimpleLogger(nil)
    }
    if cfg.DeviceName == "" {
        cfg.DeviceName = "SpotControl"
    }
    if cfg.DeviceType == 0 {
        cfg.DeviceType = spotcontrol.DeviceTypeComputer
    }
    return cfg
}
```

### Credential Selection Logic

When `Connect()` is called, credentials are selected in this order:

1. **Stored credentials** â€” If `StatePath` is set and the file contains a valid `Username` and `StoredCredentials`, those are used.
2. **Interactive login** â€” If no stored credentials exist and `Interactive` is `true`, an OAuth2 PKCE flow is initiated.
3. **Error** â€” If neither option is available, `Connect` returns an error.

## Session Options

For full control over the session, use `session.NewSessionFromOptions` directly:

```go
sess, err := session.NewSessionFromOptions(ctx, &session.Options{
    Log:         spotcontrol.NewSimpleLogger(nil),
    DeviceType:  spotcontrol.DeviceTypeComputer,
    DeviceId:    "a1b2c3d4e5f6a1b2c3d4a1b2c3d4e5f6a1b2c3d4",
    DeviceName:  "MyApp",
    Credentials: session.InteractiveCredentials{CallbackPort: 0},
    ClientToken: "",            // auto-fetched if empty
    Resolver:    nil,           // auto-created if nil
    Client:      nil,           // default: 30s timeout
    AppState:    loadedState,   // optional: restores OAuth2 token
})
```

### Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `Log` | `spotcontrol.Logger` | No | `NullLogger` | Logger instance for structured logging throughout the session. |
| `DeviceType` | `devicespb.DeviceType` | **Yes** | â€” | The Spotify device type. Use the constants from the `spotcontrol` package (e.g. `spotcontrol.DeviceTypeComputer`). Must not be `UNKNOWN`. |
| `DeviceId` | `string` | No | Auto-generated | 40-hex-char device identifier. If empty, a random one is generated via `GenerateDeviceId()`. If provided, must be exactly 40 valid hex characters (20 bytes). |
| `DeviceName` | `string` | No | `"SpotControl"` | Human-readable name shown in Spotify Connect device lists. |
| `Credentials` | `session.Credentials` | **Yes** | â€” | Authentication credentials. Must be one of: `StoredCredentials`, `SpotifyTokenCredentials`, `BlobCredentials`, or `InteractiveCredentials`. |
| `ClientToken` | `string` | No | Auto-fetched | An existing client token string. If empty, a new one is retrieved from `https://clienttoken.spotify.com/v1/clienttoken`. |
| `Resolver` | `*apresolve.ApResolver` | No | Auto-created | An existing AP resolver instance. If `nil`, a new one is created using the default `https://apresolve.spotify.com/` endpoint. |
| `Client` | `*http.Client` | No | 30s timeout | HTTP client used for all HTTP requests (Login5, spclient, apresolve, client token, OAuth2). If `nil`, a default client with a 30-second timeout is created. |
| `AppState` | `*spotcontrol.AppState` | No | `nil` | Previously persisted application state. If provided and contains an OAuth2 token (access token or refresh token), that token is restored so Web API requests work without a new interactive login. |

### Validation

- `DeviceType` must not be `UNKNOWN` (zero value) â€” returns an error otherwise.
- `DeviceId`, if non-empty, must be exactly 40 valid hex characters â€” returns an error otherwise.
- `Credentials` must be one of the four supported types â€” returns an error for unknown types.

## Controller Configuration

The controller is configured via `controller.Config` when using `controller.NewController` directly:

```go
ctrl := controller.NewController(controller.Config{
    Log:            spotcontrol.NewSimpleLogger(nil),
    Spclient:       sess.Spclient(),
    Dealer:         sess.Dealer(),
    DeviceId:       sess.DeviceId(),
    DeviceName:     "MyApp",
    DeviceType:     spotcontrol.DeviceTypeComputer,
    UseWebApi:      false,
    VolumeDebounce: 500 * time.Millisecond,
})
```

### Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `Log` | `spotcontrol.Logger` | No | `NullLogger` | Logger instance. |
| `Spclient` | `*spclient.Spclient` | **Yes** | â€” | The spclient HTTP wrapper (from `Session.Spclient()`). |
| `Dealer` | `*dealer.Dealer` | **Yes** | â€” | The dealer WebSocket client (from `Session.Dealer()`). |
| `DeviceId` | `string` | **Yes** | â€” | This controller's device identifier. |
| `DeviceName` | `string` | **Yes** | â€” | Human-readable name for device registration. |
| `DeviceType` | `devicespb.DeviceType` | **Yes** | â€” | Spotify device type for device registration. |
| `UseWebApi` | `bool` | No | `false` | When `true`, playback commands are routed directly to `api.spotify.com` instead of through the spclient proxy. **Not recommended** â€” subject to stricter rate limits. |
| `VolumeDebounce` | `time.Duration` | No | `500ms` | Duration to debounce volume updates. Set to a negative value to disable debouncing entirely. Zero uses the default (500ms). |

### Convenience: `Session.NewController()`

For most use cases, use `Session.NewController()` which automatically configures all required fields from the session:

```go
ctrl := sess.NewController()
```

This is equivalent to:

```go
ctrl := controller.NewController(controller.Config{
    Log:        sess.log,
    Spclient:   sess.Spclient(),
    Dealer:     sess.Dealer(),
    DeviceId:   sess.DeviceId(),
    DeviceName: sess.DeviceName(),
    DeviceType: sess.DeviceType(),
})
```

## State Persistence

spotcontrol provides a simple JSON-based state persistence system for saving and restoring session data (device ID, credentials, OAuth2 tokens) across application runs.

### `AppState` Structure

```go
type AppState struct {
    DeviceId          string    `json:"device_id"`
    Username          string    `json:"username,omitempty"`
    StoredCredentials []byte    `json:"stored_credentials,omitempty"`
    OAuthAccessToken  string    `json:"oauth_access_token,omitempty"`
    OAuthRefreshToken string    `json:"oauth_refresh_token,omitempty"`
    OAuthTokenType    string    `json:"oauth_token_type,omitempty"`
    OAuthExpiry       time.Time `json:"oauth_expiry,omitempty"`
}
```

| Field | Description |
|-------|-------------|
| `DeviceId` | 40-hex-char unique device identifier. Preserving this across sessions ensures the device ID remains stable. |
| `Username` | Spotify canonical username (e.g. `"johnsmith"` or a numeric ID). |
| `StoredCredentials` | Reusable authentication credential bytes from the AP `APWelcome`. These are opaque blobs that let you authenticate without the original password. |
| `OAuthAccessToken` | OAuth2 access token for the Spotify Web API (`api.spotify.com`). Short-lived (typically 1 hour). |
| `OAuthRefreshToken` | OAuth2 refresh token for obtaining new access tokens. Long-lived. |
| `OAuthTokenType` | Token type, typically `"Bearer"`. |
| `OAuthExpiry` | When the access token expires. The refresh token is used automatically when this time passes. |

**Helper method:**

```go
func (s *AppState) HasOAuthToken() bool
```

Returns `true` if either `OAuthAccessToken` or `OAuthRefreshToken` is non-empty.

### Saving State

```go
state := sess.ExportState()
err := spotcontrol.SaveState("state.json", state)
```

`SaveState`:
1. Marshals the `AppState` as pretty-printed JSON (with 2-space indentation)
2. Writes to the specified path with file mode `0600` (owner read/write only)
3. Returns an error if marshaling or writing fails

### Loading State

```go
state, err := spotcontrol.LoadState("state.json")
```

`LoadState`:
1. Reads the file at the specified path
2. If the file **does not exist**, returns `(nil, nil)` â€” this is *not* an error
3. If the file exists but cannot be read or parsed, returns `(nil, error)`
4. If successful, returns `(*AppState, nil)`

**Typical usage pattern:**

```go
state, err := spotcontrol.LoadState("state.json")
if err != nil {
    log.Fatal(err)  // Read/parse error
}
if state == nil {
    // No prior state â€” first run
    // Use InteractiveCredentials
} else {
    // Use StoredCredentials from state
}
```

### State File Format

The state file is plain JSON:

```json
{
  "device_id": "a1b2c3d4e5f6a1b2c3d4a1b2c3d4e5f6a1b2c3d4",
  "username": "spotify_user",
  "stored_credentials": "base64_encoded_bytes...",
  "oauth_access_token": "BQD...",
  "oauth_refresh_token": "AQB...",
  "oauth_token_type": "Bearer",
  "oauth_expiry": "2025-06-15T12:00:00Z"
}
```

Notes:
- `stored_credentials` is base64-encoded (Go's `json.Marshal` encodes `[]byte` as base64)
- Fields with `omitempty` tags are omitted when empty/zero
- Times are in RFC 3339 format

### File Permissions

`SaveState` writes with mode `0600` â€” only the file owner can read or write. This is important because the file contains:

- **Stored credentials** â€” equivalent to a password for Spotify authentication
- **OAuth2 tokens** â€” can be used to make API requests on behalf of the user

If you need additional security, wrap `SaveState`/`LoadState` with your own encryption layer.

### State Lifecycle with `quick.Connect()`

When using `quick.Connect()` with a `StatePath`:

1. **Load**: State is loaded from `StatePath` at the beginning
2. **Credential selection**: If stored credentials exist, they're used; otherwise interactive login
3. **Session creation**: The `AppState` is passed to `session.Options.AppState` to restore the OAuth2 token
4. **Save**: After successful session creation, `ExportState()` is called and saved back to `StatePath`

This means the state file is updated on every successful connection, ensuring credentials and tokens stay fresh.

## Logging

spotcontrol defines a `Logger` interface used throughout all packages. You can use one of the three provided implementations or write your own.

### Logger Interface

```go
type Logger interface {
    Tracef(format string, args ...interface{})
    Debugf(format string, args ...interface{})
    Infof(format string, args ...interface{})
    Warnf(format string, args ...interface{})
    Errorf(format string, args ...interface{})

    Trace(args ...interface{})
    Debug(args ...interface{})
    Info(args ...interface{})
    Warn(args ...interface{})
    Error(args ...interface{})

    WithField(key string, value interface{}) Logger
    WithError(err error) Logger
}
```

The interface is compatible with logrus-style structured loggers.

### Log Levels

| Level | When Used |
|-------|-----------|
| **Trace** | Extremely verbose: packet-level details, ping/pong messages, recv loop events |
| **Debug** | Detailed operational info: token renewal, key exchange completion, connection state changes |
| **Info** | Significant events: session established, device registered, authentication success |
| **Warn** | Recoverable issues: connect-state fallback to Web API, missing OAuth2 token, dropped events |
| **Error** | Failures: connection loss, failed requests, unmarshal errors |

### SimpleLogger

Writes human-readable log lines to an `io.Writer`. Trace-level messages are suppressed.

```go
// Write to stderr (default)
log := spotcontrol.NewSimpleLogger(nil)

// Write to stdout
log := spotcontrol.NewSimpleLogger(os.Stdout)

// Write to a file
f, _ := os.Create("spotcontrol.log")
log := spotcontrol.NewSimpleLogger(f)
```

**Output format:**

```
[INF] session established for user joh***
[DBG] obtained new client token: 65b70807...87bd
[WRN] connect-state play playlist failed, falling back to Web API: ...
[ERR] failed sending debounced volume command
```

**Characteristics:**
- Trace messages are silently discarded
- `WithField` prefixes subsequent messages with `[key=value]`
- `WithError` is equivalent to `WithField("error", err.Error())`
- Thread-safe (writes are atomic per log call)

### SlogLogger

Adapts Go's standard `log/slog.Logger` to the spotcontrol `Logger` interface.

```go
import "log/slog"

// JSON output to stderr at debug level
handler := slog.NewJSONHandler(os.Stderr, &slog.HandlerOptions{
    Level: slog.LevelDebug,
})
slogger := slog.New(handler)
log := spotcontrol.NewSlogLogger(slogger)
```

```go
// Text output with custom options
handler := slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{
    Level:     slog.LevelInfo,
    AddSource: true,
})
log := spotcontrol.NewSlogLogger(slog.New(handler))
```

**Level mapping:**

| spotcontrol Level | slog Level |
|-------------------|------------|
| Trace | Debug - 4 (slog does not have a built-in Trace level) |
| Debug | `slog.LevelDebug` |
| Info | `slog.LevelInfo` |
| Warn | `slog.LevelWarn` |
| Error | `slog.LevelError` |

### NullLogger

Discards all log output. Useful for testing or when you want complete silence.

```go
log := &spotcontrol.NullLogger{}
```

All methods are no-ops. `WithField` and `WithError` return the same `NullLogger` instance.

### Custom Logger

Implement the `Logger` interface to integrate with your own logging framework (e.g. zerolog, zap, logrus):

```go
type MyLogger struct {
    // your fields
}

func (l *MyLogger) Tracef(format string, args ...interface{}) { /* ... */ }
func (l *MyLogger) Debugf(format string, args ...interface{}) { /* ... */ }
func (l *MyLogger) Infof(format string, args ...interface{})  { /* ... */ }
func (l *MyLogger) Warnf(format string, args ...interface{})  { /* ... */ }
func (l *MyLogger) Errorf(format string, args ...interface{}) { /* ... */ }

func (l *MyLogger) Trace(args ...interface{}) { /* ... */ }
func (l *MyLogger) Debug(args ...interface{}) { /* ... */ }
func (l *MyLogger) Info(args ...interface{})  { /* ... */ }
func (l *MyLogger) Warn(args ...interface{})  { /* ... */ }
func (l *MyLogger) Error(args ...interface{}) { /* ... */ }

func (l *MyLogger) WithField(key string, value interface{}) spotcontrol.Logger {
    // Return a new logger with the field added to context
    return &MyLogger{ /* ... */ }
}

func (l *MyLogger) WithError(err error) spotcontrol.Logger {
    return l.WithField("error", err)
}
```

### Username Obfuscation

All built-in log messages that include usernames use `ObfuscateUsername()` for safety:

```go
spotcontrol.ObfuscateUsername("johnsmith")  // â†’ "joh***"
spotcontrol.ObfuscateUsername("ab")         // â†’ "***"
spotcontrol.ObfuscateUsername("")           // â†’ "***"
```

This ensures usernames are never fully logged, even at debug level.

## Device Configuration

### Device ID

The device ID is a 40-character lowercase hexadecimal string (20 random bytes), uniquely identifying this controller to Spotify.

**Auto-generation:**

```go
// Let the session generate one
sess, _ := session.NewSessionFromOptions(ctx, &session.Options{
    DeviceId: "",  // auto-generated
    // ...
})
fmt.Println(sess.DeviceId())  // e.g. "a1b2c3d4e5f6a1b2c3d4a1b2c3d4e5f6a1b2c3d4"
```

**Manual generation:**

```go
deviceId := spotcontrol.GenerateDeviceId()
// Uses crypto/rand â€” cryptographically secure
```

**Persistence:**

The device ID should be persisted (via `AppState`) so that the same device identity is maintained across sessions. If a new device ID is generated each time, Spotify will show a new device in the Connect list on every run.

**Validation rules:**
- Must be exactly 40 hex characters (20 bytes)
- Must be valid hexadecimal (0-9, a-f)
- The session validates this and returns an error if invalid

### Device Name

The human-readable name shown in Spotify Connect device lists on other Spotify clients (e.g. the phone app, desktop client).

```go
// Via QuickConfig
quick.QuickConfig{DeviceName: "Living Room Controller"}

// Via session options
session.Options{DeviceName: "Living Room Controller"}
```

Default: `"SpotControl"`

The name can be any string. It does not need to be unique.

## HTTP Client

An `*http.Client` can be provided to `session.Options` for control over HTTP behavior:

```go
client := &http.Client{
    Timeout: 60 * time.Second,
    Transport: &http.Transport{
        TLSClientConfig: &tls.Config{
            MinVersion: tls.VersionTLS13,
        },
        MaxIdleConns:        10,
        IdleConnTimeout:     90 * time.Second,
    },
}

sess, _ := session.NewSessionFromOptions(ctx, &session.Options{
    Client: client,
    // ...
})
```

The same HTTP client is used for:
- Client token requests (`clienttoken.spotify.com`)
- AP resolver requests (`apresolve.spotify.com`)
- Login5 requests (`login5.spotify.com`)
- Spclient requests (connect-state, metadata, Web API proxy)
- Dealer WebSocket upgrade (the `Transport`, `CheckRedirect`, `Jar`, and a 10-second `Timeout` are inherited)
- OAuth2 token exchange (`accounts.spotify.com`)

If `nil`, a default client with a 30-second timeout is created.

## Volume Debounce

Volume updates are debounced to prevent flooding Spotify's backend during rapid adjustments (e.g. holding a slider).

### Default Behavior

500ms debounce delay (matching librespot's `VOLUME_UPDATE_DELAY`):

```go
ctrl := controller.NewController(controller.Config{
    VolumeDebounce: 0,  // 0 means use default (500ms)
    // ...
})
```

### Custom Duration

```go
ctrl := controller.NewController(controller.Config{
    VolumeDebounce: 200 * time.Millisecond,  // faster response
    // ...
})
```

### Disable Debouncing

```go
ctrl := controller.NewController(controller.Config{
    VolumeDebounce: -1 * time.Millisecond,  // any negative value disables
    // ...
})
```

With debouncing disabled, every `SetVolume()` call sends a request immediately.

### How It Works

1. `SetVolume()` is called â€” the volume value and target device are stored
2. A timer is started (or reset if already running) for the debounce duration
3. If `SetVolume()` is called again before the timer fires, the stored values are updated and the timer resets
4. When the timer fires, the most recent volume value is sent via the connect-state volume endpoint
5. If the send fails, an error is logged but not returned (since the original `SetVolume()` call already returned)

## Command Routing

### Default: Spclient Proxy

By default, the controller routes playback commands through two paths:

1. **Connect-state endpoints** (preferred) â€” player commands, volume, transfer
2. **Spclient proxy** (fallback) â€” Web API style `/v1/me/player/*` paths routed through the spclient infrastructure

This is the recommended configuration because it avoids the stricter rate limits on `api.spotify.com`.

### Web API Mode

Setting `UseWebApi: true` forces all commands directly to `api.spotify.com`:

```go
ctrl := controller.NewController(controller.Config{
    UseWebApi: true,  // NOT recommended
    // ...
})
```

**Reasons to avoid this:**
- `api.spotify.com` has stricter rate limits (HTTP 429 responses)
- Requires a valid OAuth2 token (Login5 tokens may lack the required scopes)
- Some operations may be slower or less reliable
- The connect-state protocol provides a richer feature set

**When to use this:**
- Debugging â€” to compare behavior between the two paths
- If the spclient proxy is unreachable in your network environment

## OAuth2 Callback Port

When using `InteractiveCredentials`, a local HTTP server is started to receive the OAuth2 authorization code callback.

### Random Port (recommended)

```go
session.InteractiveCredentials{
    CallbackPort: 0,  // OS chooses a random available port
}
```

The actual port used is printed in the authorization URL. This is the safest option as it avoids port conflicts.

### Fixed Port

```go
session.InteractiveCredentials{
    CallbackPort: 8080,  // Use port 8080
}
```

Useful if you need a predictable callback URL (e.g. for firewall rules or a pre-configured Spotify app redirect URI).

### Callback URL Format

The callback server listens on:

```
http://127.0.0.1:{port}/login
```

The OAuth2 redirect URL is constructed as `http://127.0.0.1:{port}/login` and included in the authorization request. After the user authenticates in their browser, Spotify redirects to this URL with the authorization code.

## Version & User Agent

spotcontrol identifies itself in several places using version information:

### User-Agent Header

All HTTP requests include:

```
User-Agent: spotcontrol/<version> Go/<go-version>
```

Example: `spotcontrol/1.0.0 Go/go1.23.0`

### AP Key Exchange

The `ClientHello` message includes:

| Field | Value |
|-------|-------|
| Product | `PRODUCT_CLIENT` |
| ProductFlags | `PRODUCT_FLAG_DEV_BUILD` (if version is `"dev"`) or `PRODUCT_FLAG_NONE` |
| Platform | OS/arch-specific (e.g. `PLATFORM_OSX_X86_64`, `PLATFORM_LINUX_X86_64`) |
| Version | `127700358` (the `SpotifyVersionCode` constant) |

### Device Registration

The device's `DeviceSoftwareVersion` field is set to `"spotcontrol/<version>"`.

### Version Sources

The version string is determined at build time:

1. **ldflags**: If set via `-ldflags "-X github.com/mcMineyC/spotcontrol.version=1.0.0"`, that value is used
2. **Git commit**: If built with Go module VCS info, the first 8 characters of the commit hash are used
3. **Fallback**: `"dev"`

## Platform Detection

spotcontrol automatically detects the platform for protobuf messages sent during authentication and client token retrieval. The detection is based on `runtime.GOOS` and `runtime.GOARCH`.

### Supported Platforms

| GOOS | GOARCH | Platform Enum | Client Token Data |
|------|--------|---------------|-------------------|
| `darwin` | `amd64`/`arm64` | `PLATFORM_OSX_X86_64` | `NativeDesktopMacOSData` |
| `linux` | `amd64` | `PLATFORM_LINUX_X86_64` | `NativeDesktopLinuxData` |
| `linux` | `arm`/`arm64` | `PLATFORM_LINUX_ARM` | `NativeDesktopLinuxData` |
| `windows` | `amd64` | `PLATFORM_WIN32_X86_64` | `NativeDesktopWindowsData` |
| `windows` | `386` | `PLATFORM_WIN32_X86` | `NativeDesktopWindowsData` |
| `freebsd` | `amd64` | `PLATFORM_FREEBSD_X86_64` | `NativeDesktopLinuxData` |
| `android` | any | `PLATFORM_ANDROID_ARM` | `NativeAndroidData` |
| `ios` | `arm64` | `PLATFORM_IPHONE_ARM64` | `NativeIOSData` |

Unsupported combinations fall back to `PLATFORM_GENERIC_PARTNER` and `nil` platform-specific data (which may cause issues with the client token endpoint).

## Environment & Runtime Notes

### Go Version

spotcontrol requires **Go 1.23** or later (as specified in `go.mod`).

### Thread Safety

All spotcontrol types are safe for concurrent use from multiple goroutines:

- `Session` â€” immutable after creation (except OAuth2 token, which is mutex-protected)
- `Controller` â€” all state is mutex-protected; event channels are synchronized
- `Dealer` â€” connection state and receiver lists are mutex-protected
- `Accesspoint` â€” send/receive have independent locks
- `Mercury.Client` â€” pending requests and subscriptions are mutex-protected
- `Login5` â€” token state is mutex-protected
- `ApResolver` â€” endpoint cache is mutex-protected
- `Spclient` â€” stateless per-request (token functions are already thread-safe)

### Graceful Shutdown

The recommended shutdown pattern:

```go
ctx, cancel := context.WithCancel(context.Background())
defer cancel()

// Handle signals
sigCh := make(chan os.Signal, 1)
signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
go func() {
    <-sigCh
    cancel()
}()

result, err := quick.Connect(ctx, quick.QuickConfig{...})
if err != nil { log.Fatal(err) }
defer result.Close()

// Your application logic...
// When ctx is cancelled (via signal), clean up and exit
```

The `Close()` methods handle:
1. Stopping background goroutines
2. Closing WebSocket connections
3. Closing TCP connections
4. Closing all event subscriber channels
